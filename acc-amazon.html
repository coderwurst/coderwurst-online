Accessibility Study: Amazon Germany

5 Minute Read including: ux, accessibility
27.11.2018



Accessibility; what's done right

  • Good use of ARIA Attributes for UI components that cannot be effectively made accessible using only html, for example, the search bar at the top of the page
  • Hidden Header element is present, making page headers availble to screen readers to allow for more comfortable navigation
  • Form label hidden but available for shopping cart to inform users of number of items in cart
  • With exception of a h1 element, page has a good semantic structure with use of h2, layout tables, lists and frames to appropriately groupd presented information

Accessibility; potential problems

  • Missing Alt Text making visual content not available to screen readers
  • Empty href attributes on image links, hiding the intended function of these elements
  • "Alt=Alt" Text provides no meaningful information to a screen reader user, and is made more confusing as several of these images are grouped and will therefore be read together
  • Missing visible first level heading (H1) on home page presents not only problems with the screen reader, but could also harm your sites Google Ranking
  • Missing Tab Indexes on fields can make it hard for keyboard only users to sucessfully navigate the site
