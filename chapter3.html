chapter 3 - programming with Java operators and strings

Java Operators and Strings

  • these are the most fundamental elements of the Java language
  • Strings are commonly used in Java and throughout the exam

Fundamental Operators
  • used to return a result from an expression using 1 - 3 operands
bestCoins = goldCoins++; // has 1 operand
totalCoins = silverCoins + goldCoins // has 2 operands
int pirateShares = (isCaptain)?TEN_SHARES : FIVE_SHARES; // 3 operands (only used with ?:)


Assignment Operator
  • used to assign values
  • '='
  • moves valid literals into variables
  • compilation errors occur where an invalid literal is assigned to a variable of an associated type
  • examples of compilation errors include using TRUE instead of true when assigning a boolean, using a value greater than 127 when assigning a byte value, and when an invalid parameter is passed into an object constructor upon initiation
  • compound assignment operators included in the exam are += and -= which adds or subtracts a literal value to the current variable value
goldCoins = goldCoins + 99; is the same as goldCoins += 99;
  • the advantage of using compound assignment is the reduction of keystrokes, however it is generally not used as the long hand versions are more readable and there create more maintainable code

Arithmetic Operators
  • 9 in total covered in the exam; 5 basic maths operators for +, -, *, / and %
  • 4 for prefix increment ++aValue, prefix decrement --aValue, postfix increment aValue++, postfix decrement aValue--
  • the prefix operators allow a shorthand way of incrementing or decrementing a variable by 1
  • postfix allows for incrementation and decrementation after the value has been evaluated
  • an example; y = aValue++; sets y to aValue before incrementing aValue by 1
  • y=aValue++; means that y = aValue and aValue is then aValue++;
  • y=++aValue; means that y= the newly incremented aValue (aValue +1)

Relational Operators
Basic Relational Operators
  • return boolean values in relation to their left and right operands
  • <, >, <=, >=
  • most commonly used with Integer operands, eg. 1 < 2 BUT can also be used with Chars
  • A is less than 2, therefore A < B is true, as well as the other character evaluations of A and B including unicode, hexidecimal, octal etc.
  • the same function can be performed with floating point numbers with d, D, f or F postfixes
  • values such as floating points can be cast as doubles before being compared
Equality Operators
  • == to test if operand left is equal to operand right, or != for not equal to
  • can test primitive data types, as well as objects
  • when testing objects, the == operator tests to see if the reference is the same
Object a = new Object();
Object b = new Object();
Object c = b; 
a != b; // is true b != a; // is also true c == b; // is true

  • if the operands being compared are of different primitive types, BINARY NUMERIC PROMOTION are applied...
  • double -> float -> long -> when all else fails, int!!!

Logical Operators
  • 3 types; logical AND, logical OR and logical negation

Logical (Conditional) Operators && ||
  • AND OR statements evaluate their left and right operands, if both are true, they return true
  • if an AND && Operator evaluates the left operand and finds it returns false, it doesn't evaluate right operand as the return can only be false - known as a short-circuit operator
  • can also be used to assign a boolean value; boolean myAnswer = operand1 && operand2; // will assign true or false depending on the results of the left and right operand evaluation
  • an OR || operator will return true if either the left or right operator is true - again if the left operand is evaluated and found to be true, the process terminates, and the right operand is not evaluated. Instead the true value is returned; also known as a short-circuit operator
Logical Invertion Operator
  • a.k.a.Boolean invert operator
  • used to return the opposite of a boolean value
  • for example, !false; returns true
  • !!true; returns true
  • !!!true; returns false
  • !!!!true; returns true
  • logical invertion operator cannot be used on a non-boolean value, for example !STRING will not compile. However, (!"STRING".equal("STRING")); would compile and compute

Understanding Operator Precedence
  • the order which operators will be evaluated when several operators are found in the same expression
  • the higher precedence operator will always be evaluated first
  • operator precedence can be overridden with the use of parentheses
  • (1+2+3) is equal to 6, (1+2*3) is equal to 7, but ((1+2)*3) is equal to 9

Using String Objects and Their Methods
  • used to represent 16-bit unicode character strings (bits for each character added onto the bits for the next character)
  • strings are IMMUTABLE - cannot be changed
  • string.replace("something", "another thing") will not change the contents of string, instead a new variable, eg string2, needs to be created
  • String class - creates an immutable string object, eg. String myString = ""; or String myString = new String (); or String myString = new String ("");
  • StringBuilder - creates a mutable string object
  • StringBuffer - creates a mutable, thread safe String object
  • a previous string held in a reference variable can, however, be replaced with a new string
String Concatenation
  • + used to join strings together
  • functions on left associative...
x = 5, y = 5;
sout(x + y + " is the total"); 
-> 10 is the total; nothing on left hand side of ints to associate sout((x + y) + " is the total");
-> 10 is the total; nothing on left hand side of ints to associate, parentheses used to seperate out x + y sout("the total is " + (x + y)); 
-> the total is 10; sting on left hand side of ints to associate, but parentheses used to seperate out x + y sout "the total is " + x + y)); 
-> the total is 55; no parentheses used to seperate out x + y, therefore the x and y values are concatenated

the toString Method
  • all classes extend this class by default, therefore all inherit this method
  • uses the String concatenation operator
  • often overridden to return the most appropriate information needed to describe the object
  • must be careful when printing out integers that don't have a String as an initial value
eg. 
String title1 = "shovels.";
String title2 = "Shovels:";
int flatShovels = 5;
int roundShovels = 6; 

Sout(flatShovels + roundShovels + title1); // 11 Shovels Sout((flatShovels + roundShovels) + titls1);
// 11 Shovels Sout(title2 + flatShovels + roundShovels);
// Shovels: 56 Sout(title2 + (flatShovels + roundShovels)); // Shovels: 11

Methods of the String Class
The charAt Method
  • returns a Char primitive value from the specified int index
  • note that indexes, as with arrays, starts at 0
  • public char charAt(int index) char c1 = string.charAt(3);

The indexOf Method
  • returns a primitive int representing the index of a char or string in a relationship to the referenced string object
  • .indexOf('u'); ..indexOf('u', 9) // starts from index 9
  • the above examples can also be used with Strings
  • if no occurrence is found, the method returns a -1
The length Method
  • returns a primitive int value representing the number of characters in a given String
  • WOFIE - string method for length uses paranthesese .length() whereas arrays use an instance variable in their state, array.length; somethign to look out for in exam
The concat Method
  • can be used to add another string onto a previous string
  • myString.concat(string2);
The replace Method
  • .replace(char oldChar, char newChar) or .replace(CharSequence string1, CharSequence string2)
  • replaces the old with the new
  • CharSequence interface allows for the use of String, StringBuffer or StringBuilder objects
The startsWith Method
  • can be used to determine if a String starts with a defined String
  • the start point of the evaluation can also be offset
  • returns a boolean value
  • .startsWith(String prefix) or .startsWith(String prefix, int offset)
The endsWith method
  • similar to startsWith, but only one possible method declaration
  • to determine if a string ends with a defined suffix
  • .endsWith(String suffix)
The subString method
  • used to return a snippet of a particular string
  • ints declared as the start and end point of the substring
  • .subString(9) will cut out a string from index 9 to the end .subString(9,10) will cut out a single character at index 9 .subString(9, 23) will cut out from index 9 - 23 .subString(9,8) is out of range .subString(9,9) will return a blank
The trim Method
  • used to remove leading or trailing white spaces from strings
the toLowerCase and toUpperCase Methods
  • will convert a string to its lower or upper case equivalent
string.toLowerCase(); or string.toUpperCaselocale locale) can be used
The equalsIgnoreCase Method
  • compares contents of 2 strings and returns true if contents match, ignoring any case differences
  • string1.equalsIgnoreCase("this String");

Chaining
  • method1.method2.method3 can be used to perform a number of String methods in the same line of code
  • after chained method calls, it is also possible to concatenate the newly formed string, eg. + "."
  • string.trim().replace("string1", "string2").substring(0,43) + ".";

Using StringBuilder Objects and Their Methods
  • StringBuilder class represents a mutable character string
  • StringBuffer represents a mutable, thread-safe character string

Most common StringBuilder methods:

The append Method

  • appends sullplied data as a character string
  • can be used to append String, Object, StringBuffer, charSequence, char, boolean, int, long, float and doubles
  • but all converted to char types in the final string
The insert Method
  • allows you to insert a string representation of the supplied data at a specified point
  • string.insert(int offset, String myString);
The delete Method
  • removes characters in a substring
  • string.delete(int start, int end);
The deleteCharAt Method
  • removes the character at the specified index
  • string.deleteCharAt(11);
The reverse Method
  • string.reverse();

== || .equals()
  • when comparing char Strings between 2 strings, the .equals() should be used
  • == returns true if the REFERENCES are the same, not the content

String string2 = myString; // using == for the above strings would return true!
