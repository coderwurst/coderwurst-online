chapter 2 - programming with Java statements

Java Statements
  • the language allows the proper sequence of executions with associated functionality to occur
  • the more statement types a language has, the more effective a language can be
  • Java allows the system to continue to run even if an exception is thrown by the inclusion of handling statements
  • The Java Language Specification contains all Java statements
  • an effective Java programmer must master the basic statements - all of which are covered in the exam
    • On the exam:break case continue while do-while expression for loop enhanced for loop if if-then if-then-else return switch throw try-catch-finally

    Understand Assignment Statements
    • sets a value within a variable
    • considered to be expression statements (anything that has a value or is reduced with expressions)
    • inc are adding numbers, concatenating strings, etc.
    • all expression statements must end with a semi-colon
    • assignment expression statements are designed to assign values to variables
    • variable = value;

    Expression statements
    • pre-increment ++variable
    • pre-decrement --variable
    • post-increment variable++
    • post-decrement variable--

    • a value assigned to a variable will remain stored in memory unless changed by another expression statement, or it is killed

    Create and Use Conditional Statements
    • control flow based on conditions
    The if statement
    • conditionally execute a piece of code, or decide between a choice of statements
    • braces used for multiple statements to determine the beginning and end of each if { }
    • else clause is optional, used to ensure that some code gets executed
    • if - else statements function with boolean values
    • switch statements can be used with most other primitive value types
    • an if statement does not necessarily require an else branch
    • and if-then statement must have at least one else if branch
    • an if-then-else statement is a if-then which finishes on an else (else-if in this case are not optional)
    • can accept any expression that returns a boolean value
    • if the first condition is met and an else statement is present, the else statement will be skipped
    • (boolean) and (!boolean) can be used
    • (boolean == false) and (boolean = false) can also be used

    The if-then conditional statement
    • a.k.a. if-else-if
    • used to include a multiple of conditions in the flow
    • see conditional example in IntelliJ
    • NB - debug does not stop on variable initiation
    • although an if expression requires a boolean value, an object from the BOOLEAN WRAPPER class is also allowed, as it will go through unboxing in order to return the expected type
    • UNBOXING is the automatic production of its primitive value
    • more to come in Chapter 4

    The if-then-else conditional statement
    • the final else branch of this kind of statement means that there will always be a default piece of code for exeecution
    • abrupt termination occurs, for example, due to an overflow error - in this event all subsequent if-else-if and else branches will not be executed

    The Switch Conditional Statement
    • compares a value in the switch statement expression with the value stored in a case keyword
    • once matched, the enclosed statements are executed
    • NB that all subsequent statements will also be executed, unless a break statement is included in the affected case statement
    • a break statement causes immediate termination of a switch statement
    • primitive value types that can be used in a switch condition inc; byte, short, int or char
    • Wrapper classes are also unboxed and used, same as if statements
    • Enumerations are also allowed, and as of Java 7, String expressions
    • the default switch option will be executed if no other case value is found
    • to ensure a statement is bug free, break and default statements are necessary to prevent improper fall throughs and unhandled conditions
    • NB that unlike some other languages, using ranges in case statements is NOT ALLOWED - use an if statement instead
    • also NB String conditions are only available in switch statements from Java 7, pre versions must use an if equivalent

    Create and Use Iteration Statements
    The for loop
    • (initialisation part; expression; iteration)
    • initialisation does not need to declare a variable as long as it has been declared within the scope of the loop
    • the variable can also be reset in the for loop (back to 0 for example)
    • as soon as the for loop ends the scope for this variable also ends - no longer available
    
for (int m=1; m < 10; m++) {// sequence of events}sout("value: " + m);
    
this code will not throw a complication error as the m value is out of scope once the foor loop has been left
    • the expression value must evaluate to a boolean
    • the iteration (a.k.a.) update part, provides the mechanism to update the expression; eg. i++, i--


    NB that compiler error messages can be different depending on the actual Oracle JDK and IDE versions found in Eclipse, etc. Look at differences - look up differences between Eclipse and javac

    The Enhanced for loop iteration statement
    • used to iterate through an array, a collection or any object that implements the interface, iterable
    • iteration occurs for each element in the array or iterable class
    • iterations can be terminated at any time with the use of a break; statement
    • the continue statement will terminate the current iteration and start with the next iteration
    for (variable : collection) { }
    • involves much less code than traditional for loop, also ensures every element is printed out
    • frequently used for searching through elements in a collection

    NB that when adding floats as above the f is needed to distinguish it as a float and not a double!!!
    Possible to use http://jalopy.sourceforge.net/ when using an IDE to format the source code

    The while and do-while iteration statements
    • a while loop will iterate as long as the expression is true
    while (expression) { }
    • at some point in the body of the loop, the expression should be updated to finish the loop operation, otherwise it could create an infinite loop
    • do - whiles are similar, only that the statement is preformed at least 1 time
    • the loop then continues to be executed as long as the while condition is true
    do { } while (expression)

    Selecting the right statements

    Create and Use Transfer of Control Statements
    The break transfer of control statement
    • used to exit or force an abrupt termination of the body of a switch statement
    • can also be used for a do, for, advanced for and do-while
    example within a for loop
    if (fishingHours > allowedFishingHours) { break;}

    The continue transfer of control statement
    • terminates the current iteration of a do, for, while, do - while and move onto the next iteration
    for (int i = 0; i < 8; i++ ) { 
sout ("day " + i + " camping ");
totalDaysCamping++;

if (totalDaysCamping > daysAllowedFishing) { 
continue; 
} 
sout (" and fishing");
}

    The return transfer of control statement
    • used to exit a method, optionally return a specified value as an expression
    • can be used to directly return aValue; or return oneValue + anotherValue; etc.
    The labeled Statement
    • not used very often
    • used to give a statement a prefixed label
    • usually in conjunction with the continue and break statement
    labelIdentifier: Statement (expression, etc) break labelIdentifier; || continue labelIdentifier;

    • note that the while loop 4 is unreachable due to the location of the continue statement
    • the above code also creates an infinite loop
