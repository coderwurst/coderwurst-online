progressive enhancement

22nd May , 2015 by coderwurst

The idea behind Progressive Enhancement is nothing new; it has been around since the mid 00s. Jeremy Keith (co-founder of Clearleft - http://clearleft.com ) sums up the theory as not a particular technique or technology, instead a robust engineering approach that provides: "a way of looking at things, a way of viewing how you build" (2014).

Progressive Enhancement is not only to do with JavaScript – anything other than HTML can be seen as an enhancement including; an interactive element that requires a mouse, as not all devices have a mouse attached; information presented visually on screen, cannot be appreciated by people with a visual impairment; or video and audio without transcripts and/ or subtitles are unsuitable for the hearing impaired.


Graceful Degradation vs. Progressive Enhancement

Since the mid 90s the web dev community based their work on the Graceful Degradation concept. Taken from the world of engineering, this approach is about providing users with the latest and greatest browsers the maximum experience (using the most modern tools available), whilst leaving older or otherwise more limited browser versions with the leftovers, with little attention being paid beyond fixing major problems. This is easily noticeable when looking at mobile devices, with functionality dropping from desktop to tablet, and then dropping again from tablet to phone.


image taken from designschack.com


It's worth to note that people running devices without JavaScript may not be responsible – perhaps the device they are using doesn't support JavaScript, or the settings have been disabled and the user has no ability to re-activate it (due to security constraints). It also isn't just to do with JavaScript being deactivated – temporary network errors, DNS l ookup failures, antivirus and personal firewall software can all have adverse effects on running JavaScript.

In 2003 at the SXSW conference in Texas, Steve Champeon and Nick Finck presented "Inclusive Web Design for the Future", unveiling a new blueprint for approaching web development – Progressive Enhancement . This approach focuses on the content instead of the browser. Such a website is defined and built from scratch so that it works for everybody, regardless of the technology available. After a baseline website with complete functionality has been achieved; the web experience can be further enhanced (progressively) for users & /or devices that have JavaScript running – this typically involves greater functionality for richer features (the bells and whistles!!).


image taken from designschack.com

Using the < em>Progressive Enhancement approach when developing a site is not only beneficial to the user experience, but it means that as the website is accessible without JavaScript, the site will continue to work in the event that a bug is introduced into the code… Other advantages include performance enhancements and greater ease with modular web design.


Adopting a Progressive Enhancement Strategy

Ascertain the Information Architecture of the site

  • Organise available content
  • Create wireframes for presentation
  • Prioritise elements which you want to display, and from this start the coding with these HTML elements in the correct order

Cascading Stylesheets (CSS)

  • Multiple, external stylesheets to improve workflow and help gain a greater consistency of presentation across the alternate media types, such as screen, print or mobile
  • The combination of multiple style sheets works well with making informed decisions about how the stylesheets are connected, improving scalability and how to hide certain layouts from problematic browsers

Unobtrusive JavaScript

  • Remove all inline event handlers and move JavaScript to linked external files, rather than using script tags
  • Conditional statements can discern the selectors to be used and in which situations they are applicable – allowing for effective management of the inter-relationship of CSS and JavaScript
  • Scripts written to take advantage of browser support, but not deny access if that support is unavailable

Progressive Enhancement Principles



image taken from alistapart.com


Step 1 – Make it work (content)

  • Websites should be built on a baseline of usable code
  • Users (and search engine spiders) should be able to understand page content without the need of any scripting
  • New pages or features should be able to work with HTML alone – no images, no CSS, no JavaScript
  • Interactive Elements only implemented with forms and server-side processing
  • Use of good semantic markup incorporating HTML5's new elements and attributes
  • Website will work in most browsers, for as many people and devices as possible including older legacy browsers and devices

Step 2 – Make it Work Better (presentation)

  • Extra layers then added to the baseline
  • Images, advanced styling, interactions, validation of data, charts and data tables
  • Use of CSS3 makes it possible for the design to 'jump off the page'

Step 3 – Add JavaScript or jQuery behaviours

  • Client side scripts added for greater functionality
  • Each progress step is a single addition – something extra that more modern browsers are capable of doing
  • Accessible and Usable experience
  • As much Content and Interactivity as possible

It is important to remember that we want our websites not just to be available for users today, but also for users in the coming days, months and years!
