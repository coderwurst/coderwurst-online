chapter 4 - working with basic classes and variables

Primitive Variables
  • most basic form of data storage in Java
  • 8 types
  • each type reserves a certain number of bits in memory

boolean char character byte int integer double short float long

  • calculations performed with primitives are much quicker than those preformed with objects
  • initialisation happens with type name; (a.k.a. instance variable)
  • once a variable is declared, it cannot be broken down into any smaller elements
  • a declared variable can be set or read, nothing else
  • no methods can be called from this variable
  • booleans default to false
  • other default values are 0
boolean Primitive
  • 1 bit value
  • true or false
  • exact size in Java is not defined, and may therefore use more or less space on different platforms
  • 2 valid literals; true and false
char Primitive
  • single 16-bit Unicode character
  • 16 bits in memory
  • min is \n0000 or 0
  • max is \uffff or 65,535
  • single quotes used when initialising literals
  • individual characters, special characters, Unicode characters, hexadecimal and octal representations are all valid literals
byte Primitive
  • normally used to store small, signed numbers up to a byte in size
  • 1 byte == 8 bits, max value of 127 and min of -128
  • defaults to 0
byte passengers = 4; // known as an implicit cast from int to byte
byte doors = (byte) 2; // explicit cast
short Primitive
  • used for small numbers
  • commonly used to save space over an int
  • 16 bits of memory, can store a 16 bit signed two's complement non-floating point number
  • max 32,767
  • min -32,768
  • again can be initialised implicitly or explicity as above
  • default is again 0
int Primitive
  • most commonly used
  • 32 bits of memory - 32 bit signed two's complement non-floating point number
  • max value 2,147,483,647
  • min value -2,147,483,648
byte wheelbase = 90;
int wBase = wheelbase; // implicit cast
char cyclinders = 'u\1234';
int cyc = cyclinders; // implicit cast from char to integer

long Primitive
  • default 0L
  • occupies 64 Bits to store a 64 bit two's complement non-floating point number
  • max 9,223,372,036,854,775,807
  • min -9,223,372,036,854,775,808
  • long thisLong = 1238574638274L
  • L of l appended to an int declares this number is a long
float Primitive
  • default value 0.0f
  • F or f appended to int number to determine floating point number
  • requires 32 bits of memory and may contain a 32 bit value
  • max 3.4e ^+38
  • min 1.4e^-45
double Primitive
  • used to store large decimal values
  • default of 0.0
  • can also be appended with d or D
  • 64 bits in memory
  • max 1.8e^308
  • min 5e^-324
  • doubles are preferred over floats as, as a float grows it can loose percision
  • char and boolean are unique, char is only primitive type that stores unsigned data, boolean true or false

Floating Point Math
  • floating point numbers loose precision the bigger they become
  • adding 2 numbers together can result in variances particularly after the decimal point
  • this can also lead to further inaccuracy when converting into a decimal number - for example, when using NumberFormat to display a currency
  • doubles are more accurate, but can still suffer from the same problems
  • for this reason, currencies should never be stored as a double or a float, instead the class BigDecimal should be used
  • BigDecimal is not so fast to process, but provides the required accuracy needed
Primitives vs. Their Wrapper Class
  • each primitive has a built in Wrapper class
  • this wrapper, can convert a primitive into an object
  • Integer, Double, Boolean, Short, Byte, Float, Long
  • autoboxing is the promotion of a primitive variable to its wrapper class
  • unboxing is the converstion to a primitive data type
  • instantiated like Objects - Intefer myInt = new Integer(5);
  • conversion can be used interchangeable - however is bad practice as using primitive data types is much more efficient allowing for faster calculations

Objects
  • almost everything in Java is an object
  • object-orientated-programming
  • internally objects use primitive data types
  • also contain methods fundamental philosophy on Java is to have object classes that can be reused over and over in the code
  • a developer writes a class - a file containing the code, tangible item
  • this class is the blueprint the JVM uses to create an object at runtime
  • many objects can be built from the one class, known as the JVM creating instances of the class
  • each instance created, is an object of the initial class
Initialising Objects
  • the new operator must be used to create objects Car thisCar = new Car(params)
  • objects that are instantiated are assigned a value of null, eg. Car oldCar; and cannot use any of the methods in this class - if it is tried, it will throw a null pointer exception
  • sometimes the restrictions on primitive data types mean that an object must be used - for example, the BigDecimal and BigInteger classes when using large numbers
  • objects should be used when a lot of similar data is to be stored
  • objects should be used to group together similar code in a distinct class that may be reused
  • very common error when working with objects is the NullPointerException
  • this is a very hard error to track down, but coding initially can help if new objects are always instantiated
  • it is also possible to perform a check to ensure that an object is not null before performing any of the objects methods
Arrays
  • series of variables
  • must be of the same type
  • easily accessible via an index
  • can be made up of primitive data types or objects
    • int [] testScore = new int [3];
  • box brackets should follow the data type - but not a must
  • the number in the brackets is the size of the array
  • array indexes are 0 based, i.e. start at 0 and work up - important to remember when looping over any array
  • arrays can also be multidimensional with more than one index, more to come

Enumerations
  • special data type
  • allows for a variable to be set to predefined constants
  • useful when there is a limited amount of options available and the variable is restricted to these known values only
  • eg. a deck of cards would have the enum suit { CLUBS, DIAMONDS, SPADES, HEARTS}
  • recent addition to Java 5.0
  • basically a simplified form of object but makes code more readable
Java Types
  • Java is strongly typed - not like JavaScript
  • once an int is declared, it can only contain an int
  • unless it is converted
Strongly Typed Variables
  • help create more reliable code
  • Java compiler will not allow dev to mismatch variables
  • only data types that are comparable with each other can be comverted
  • some data types can only be converted in one direction, eg a int can become a float, but a float cannot become an int without the loss of precision
    • float floatNum = 1.5F; int wasFloat = (int) floatNum; // results in wasFloat == 1 as 5 is truncated
  • the process is called casting as a different type
  • var thieVar = (type) otherVar;
  • incorrect casting results in a cast type error at runtime
  • only use casting when absolutely necessary
Naming Conventions
  • commonly all devs use the Oracle naming convention
  • Class Names should be a Noun, first letter capitalised along with EachInternalWord, should be as short as possible
  • variables should be descriptive, to show what variable is used for, lowercase and camel case forOtherWords
Literals
  • hard coded, value used with code
  • literal values are not variables
  • all primitive types except booleans are initialised with literal values
  • from Java 7, underscores will be able to be placed in numeric values to increase readibility, the compiler will ignore this at runtime, a good example would be credit card numbers, or for use instead of commas for larger numbers
