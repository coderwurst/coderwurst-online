chapter 5 - understanding methods and variable scope

Create & Use Methods
Using Method Syntax 5 parts
  • access modifier
  • return type
  • method name
  • parameter list
  • method body
<access mod> <return type> <method name> (<params)> { <body> } <return type> <name> () { <body> }
  • a method must not include parameters
  • must also not specify an access modifier - in this instance the default modifier is used

Access Modifier
  • used to let the JVM and compiler know what other objects can have access to this part of code
  • private is the most restrictive
  • default (package private)
  • protected
  • public
  • important to remember that every method has at least the default access modifier - even if it is not coded

Return Type
  • a popular use of methods is to return something to the code that has called them
  • return variables may be an object or a primitive
  • every method MUST declare a return type, or if nothing is to be returned, the void keyword
  • once a method declares a return variable, the method must include a return statement
  • once a return statement has been executed, the method exits - no further code after this statement will be performed...

Method Name
  • used to access the methodâ€™s functionality
  • camel case, more in Chapter 7

Parameter List
  • aways enclosed in parentheses after the method name
  • comma delimited list
  • max number of parameters of 255, though good design involves usually only passing a handful of parameters
  • parameters can include objects, enumeration types or primitives, each type is specified before the variable name

Method Body
  • main part of the method
  • contains the functionality that makes up the method
  • may only contain one line of code, to 100 lines - again good design will keep method sizes to 7 - 10 lines
  • if the method specifies a return type, the method must include a return statement that returns a matching data type as specified in the header

Making and Calling a Method
  • a class comprises of 2 things - variables and methods
  • instance variables make up the state of the object
  • methods are responsible for all actions
  • methods contained in objects, must be called from an instance of the object it is part of
    • Class myClass = new Class ();
    myClass.method(params);

Overloading a Method
  • when more than 1 method shares the same name, but has a different list of parameters
  • the param list may be composed of more or less parameters as the other method
  • useful when 1 operation can be carried out with a varying amount of info
  • when a max of 5 parameters can be entered, the other 4 can be set to defaults which will be used unless the user enters more, how many the user enters determines which overloaded method is used
  • overloaded methods are not required to have the same return types as their predecessors
  • for example, a method to add ints can be overloaded to add doubles
  • the compiler determines which overloaded method to use, depending on the parameters passed in
  • important to design overloaded methods so that they behave in a similar manner - 2 methods with the same name should be seen to be very similar, otherwise perhaps a separate method would be better implemented

Pass Objects by Reference & Value
Passing Primitives by Value to Methods
  • when a PRIMITIVE is passed into a method, a copy of the value is made and is passed into the method
  • if a method sets the value of the parameter to a different value, it has no effect on the initial variable
  • therefore the original variable passed in will not have been changed

Passing Objects by Reference to Methods
  • instead of making of a parameter when it is an object, a reference to the original object is passed
  • a reference is an internal index that represents the object
  • this results in the actual object values also being manipulated by the method

Typical exam Question to show a code example and a number of correct answers depending on if an object or a variable has been used as a parameter - easy to trip up on!

Understanding Variable Scope
  • scope is a term used to refer to a section of the code which has access to a declared variable, this can be from a few lines to an entire package
  • this helps in larger programs to prevent unintentional updating of the wrong variable, and allows for variable names to be reused in different sections of the codebase

Local Variables
  • declared inside of methods
  • commonly declared at the beginning of a method, though can be declared anywhere in code
  • scope extends to ONLY this method
  • typically used only once, or only used in 1 method
  • the borders of the scope are determines by braces { } surrounding the method body
  • a variable declared in a method body, cannot be accessed by anything outside the braces { }
  • a code block can also be created within another code block - e.g.. for loops and if statements, in this case a variable scope includes the block in which it is declared, and any further blocks within this initial block
  • therefore a variable declared in a method with nested ifs, can be used throughout the if statements
  • a variable declared in the innermost if statement, cannot be accessed by any of the surrounding statements
  • variables must only have a unique name if they are included in the same scope
  • common uses for local variables are for count variables and for a variable that the method will eventually return


Method Parameters
  • once defined, a call to a method with params must include the parameters of the specified data type (objects or variables)
  • params are passed in as arguments to the method

Instance Variables
  • declared in the class
  • remain in memory for as long as the instance of the class exists
  • scope includes the entire class and any methods contained within it
  • commonly used for object variables, which can be manipulated by more than one method in the class
  • ie. instance variables are commonly used to store information about the state of the object

An Objectâ€™s Lifecycle
  • objects are declared with data type and variable name
  • they then must be instantiated with the new operator, at this point a memory slot is allocated for the object to be stored
  • and initialised with the objects constructor, which sets all initial values for the object and prepares the object to be used
  • the next stage of the object lifecycle is the ready state, when it is ready to have its methods used, an object remains in this state as long as there is still a reference to this object in the code - this is where an object will spend most of its time
  • once there are no longer any active references to an object, it is eligible to be removed by the Java garbage collector, only after it has been removed can the memory allocated to an object be reused by another object

Create and Use Constructors
Making a Constructor
  • every class is required to have 1 constructor
  • called after an object is instantiated new (constructor)
  • a default constructor is always defined by Java, a user can create a new defined one (common)
  • a constructor must share its name with the class, including the capitalised first letter
  • possible to use all 4 access modifiers available to methods, but for the exam only the public access modifier will be covered
  • a constructor does not declare any return value

Overloading a Constructor
  • just as Java methods can be overloaded, so can constructors
  • again it must have the same name, but with a different parameter list
  • eg a bank loan class can have a constructor with default load rate values, and a second constructor to allow the user to specify a particular interest rate

Default Constructor
  • has no parameters
  • calls the superclasses constructor
  • calling the code cannot determine which version of the constructor is being used

this & super Keywords
this
  • used to refer, explicitly, to the current object
  • most commonly used when accessing instance variables
  • if a method or local variable has the same name as an instance variable, this can be used to refer explicitly to the instance variable
  • i.e. if a class has an instance variable myVariable, and a method in the class passes a parameter called myVariable, this.myVariable refers to the initial class variable, NOT the myVariable passed in as a parameter
  • if this is not used in the above case, any reference to myVariable would manipulate the parameter
  • can also be used to rewrite the class by calling a constructor from inside of a method

  • always good practice to use the this keyword when reference instance variables
  • a constructor can also use the this keyword, to refer to methods and constructors in the same object
  • the constructor or method which accepts the most parameters will then do the work
  • can also be used to call instance methods in the same object - however not often used

super
  • a.k.a. the objects superclass
  • used to access methods that have been overwritten in the current class or the constructor of a superclass
  • when the default constructor is used, it automatically calls its parent classâ€™s constructor by using super on the first line - super(param)
  • is a user defined method does not include the word super, the compiler will automatically make a call with super to the parent class, if a constructor does not exist here, a compile time error will be thrown
  • see parent and child class examples in chapter5 package
  • similar functionality is available when the super keyword is used with methods - super.methodName

Create static Methods and Instance Variables
  • a.k.a. class methods and class variables
  • both use static keyword
  • use of the keyword means that they belong to the class and not an instance of the class

Static Methods
  • belong to the class, and not to an object of the instances of a class
  • the cannot access any instance variables, nor standard methods
  • also, they cannot use this. or super. keywords
  • they can however access other static variables and methods
  • the keyword static must be placed after the access modifier
  • ClassName.staticMethod is used to call these methods (not an instance of the object such as newObject)
  • commonly used in utility methods, many examples in Java API - date classes, Math class, etc.
  • intended to perform a single task, no reason to maintain the state of their own object
  • most common static method - the main method!

Static Variables
  • every instance of a class has access to the same class variable
  • each instance also has access to its own instance variables
  • a good example is a package number assigning class
  • static variable, starts at 1000, and increments by 1 each time an instance of an object assigns a package number
  • the static int nextPackageNumber can be incremented after each package is assigned, so that the next instance of the object to assign a package number has access to the same overall variable tracking the current packages
  • static variables are said to be common between all instances of an object
  • each instance of an object has access to the value held in this one variable - good for a reference which must be help between multiple objects

Constants
  • static variables can be used to create constants
  • constants can never be changed
  • created by adding the keyword final to a class variable
  • <access mod> <static> <final> <type> <NAME> = xyz;
