chapter 6 - programming with Arrays

arrays

  • built into programming languages to handle multiple pieces of data of the same type
  • arrays can be used to store both primitive data and objects

One dimensional Arrays
  • acts as a container
  • stores a fixed number of elements
  • all of SAME data type
  • indexes start at 0, can be accessedwith myInts[0] etc
  • arrays are declared with [ ] with the data type before and the name after - normally after the data type
  • then the new operator is used to intiialise the array - before this is completed it is not considered an object
  • a number is to be placed inside the brackets to determine the size of the array
  • square brackets can be used to declare 1D arrays either after the type String [ ] or after the name String myNames [ ], both are valid HOWEVER a number is never palced inside either of these square brackets
  • the new operator is used to initialise an array, followed by the type and square brackets, this time the [ ] constains the size or length of an array
  • once an array has been assigned a length, it cannot be changed without re-initialising the array
  • an array of primitives will be initially set to 0, an array of objects will be initialised to contain null objects
  • it is also possible to irectly assign an array values after initialising it - this can be done with curley brackets
    • int [ ] alarms = {730, 900};String [] clockTypes = {"Wrist Watch", "Desk Clock", "Wall Clock"};Clock [ ] clocks = {new Clock(730), newClock(900)};
  • initialisation with curly brackets must be carried out on the same line...

Using 1D arrays
  • remember that the 1st index is 0
  • commonly used to compare strings, etc using methods such as equalsIgnoreCase()
  • also commonly used to get an array length with array.length
  • arrays can be used to cpoy data from one to the other with the arrayCopy() method - static method, part of the system class
arrayCopy()
  • 5 parameters
  • src parameter - source array
  • srcPos is the starting position
  • dest is the destination array where the data will be copied
  • destPos is the starting position in the destination array where the data is to be copied
  • length is the number of elements in the destination array
  • the destination array must also be declared as the same type
  • other array methods can be found in the Java utils class, including asList, binarySearch, copyOf, CopyOfRange, equals, fill and sort

Multi-Dimensional Arrays
  • an array with more than 1 index (possible to have even 3, 4 or more dimensions)
  • the language spec has no limits on the number of dimensions, however the JVM places a practical limit of 256
  • a multi-dimensional array is declared similarly to a 1D array, only with extra [ ] for each additional dimension
  • they are also initialised similarly to 1D arrays; with a new operator or with the values assigned directly with curly brackets
  • when using a new operator, it is only needed once at the start of however many brackets follow [ ] [ ] [ ]
  • remember that whatever is added within the curly brackets must be seperated with a comma
  • multidimensional arrays do not need to be of the same length

Using Multi-directional Arrays
  • all dimentions need to be accounted for when accessing this type of array
  • it is also possible to assign a sub-array to a multi-D array

ArrayList Object
  • part of the java.util package
  • object-orientated representation of a standard array as described above
  • can be used to store other objects including enumeration types
  • ArrayLists can be resized dynamically, and can be used to insert objects into the middle of an ArrayList

    • ArrayList <Integer> basicArrayList = new ArrayList <Integer> () ;

  • ArrayList objects cannot be used to store primitives as basic arrays can - if a primitive is needed, a wrapper object must be used as above
  • in the even that a primitive is added directly to an ArrayList, autoboxing places it into an object
  • the chevrons < > are used to indicate a generic type
  • the size of an ArrayList is based on the number of objects in it, and an exception will be thrown if the index is not valid, but additional space is automatically built into the array
  • the default constructor used to create an array list has a capacity of 10 which would return a size of 9
  • .size() is used to return the length
  • good coding practice to override the initial constructor with a size much bigger than 10 if the capacity needed is known, to reduce the need to multiple expansions which require quite a high overhead
  • also, the ensureCapacity(int) method can be called to increase the size of the ArrayList so that it has at least the specified capacity
  • when an element is inserted into the middle of an array, everything after it is shifted down by one
  • myArrayList.add(int position, object) to add to a specific location
  • also, .remove() can be used to remove an index, or to remove an Object .remove(Object o) or .remove (int index)
  • part of a group of classes that implement the collection interface

ArrayList vs StandardArrays
  • in most cases, ArrayList is likely to be easier to work with - allows for growing or shrinking, allows elements to be added or removed at a valid index, and offers many more methods
  • however, when efficency is a concern it is worth investigating the use of a basic array due to the overhead created by the expansion or shrinking of an ArrayList
  • if primitives are to be used, without a wrapper class, a basic array must be used
